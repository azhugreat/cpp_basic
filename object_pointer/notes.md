# 3.1
- 对象作为函数参数，可以按传值传递、指针传递或者按照引用传递。
- 对象作为函数的返回值；可以创建匿名对象返回，也可以返回当前函数对象或者引用。
- 对象指针作为函数的返回值；
	- 应该传入一个对象指针，并且一直使用该指针作为返回对象。
	- 绝不应该在函数中创建一个对象指针（在堆上），并且希望在外面写一个删除逻辑，否则会导致内存泄露。
- 对象引用作为函数的返回值；
	- 绝不应该在函数中创建一个对象并返回该对象。
	- 应该使用某个类中声明的，共有对象作为返回值；
	- 或者考虑传入一个对象用，并且返回该对象的引用；

# advanced topics 
- 在给函数传递参数的时候，我们希望更多的用引用而非指针。  
	- 指针有二重、三重指针，用法比引用更多。  
	- 在使用new运算符的时候只能使用指针。  
- 在传值的过程中实际上只有传值、传引用两种方法；传址、传指针是假的。
- 在《深入探索C++对象模型》的书中对此有讨论。

# 代理构造
一个构造函数可以调用另外的构造函数。   
这种特性在java中非常常见。  
在这一过程中要避免递归调用目标构造函数（ctor）。
 
```cpp
class A{

public:   

   A(): A(0){}

   A(int i): A(i, 0){}

   A(int i, int j) {

      // do something.
   }

};
```

调用流程：A() -> A(int) -> A(int, int)


# 不可变对象和类
- 不可变对象: 对象创建后，其内容不可改变，除非通过成员拷贝
- 不可变类: 不可变对象所属的类  
- 如何让以前定义的Circle类变成immutable class? 删除所有set类型函数。
- 注意指针成员的访问器。（返回值是私有数据成员的引用或者指针的情况）

- 让类成为“不可变类”
    - 所有数据域均设置为“私有”属性。
    - 没有更改器函数。
    - 也没有能够返回可变数据域对象的引用或指针的访问器。
- 不可变对象的目的：将一些需要共享和协同开发的内容设为不可变对象，  
可以防止内容被他人修改。起到类似const的作用。

# 常量表达式 const
- 由编译器执行代码得到的结果，也是在编译器得到的常量。数组的大小变量就是这类参数。
- 但是不一定所有const参数都是编译期常数，可能只是一般常数。
- 主要用于告知程序员这部分内容不会被修改。

# constexpr -- const expression												
- 在有些时候我们希望一个声明为const的变量就是一个编译器常量。
- constexpr的修饰值都是常量表达式（编译器常数）。
- 主要用于使编译器能更快的编译代码，提升性能。
- 具体实现有更复杂的用处、研究。  
consrexpr 表达式中允许包含分支循环等运算。
- 作为函数 如果其返回值可以在编译期算出来 则可以声明为编译期常量。

# assert断言 -> NDEBUG 宏
- 用于检验一个代码语言的假设是否成立。
- 在运行时判断是否可行。
- debug模式、调试模式、release模式。使用**cassert**来包含该表达式。
- 如果为假则中断程序。
- assert代码不能作为正常程序的一部分。
- assert((I > 0)&&(做什么的assert)) 是一种独特的写法。
- 高级assert ： static_assert(bool_constexpr, message),会输出message作为错误说明。
- assert用于处理永远也不应该发生的情况的处理。
